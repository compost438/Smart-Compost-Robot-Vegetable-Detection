# スマートコンポスト管理システム（Smart Compost Management System）

[English](README.md) | [日本語](README.ja.md)

Raspberry Pi 上で動作する、AI 支援型のコンポスト監視・制御システムです。リアルタイムの環境センシング、アクチュエータの自動制御、そしてコンピュータビジョンによる生ごみの解析を組み合わせて、コンポストの炭素/窒素比（C/N 比)を推定します。すべての情報はライブのデスクトップダッシュボードに表示され、クラウドデータベースに記録されます。

> 本システムは、**福岡工業大学** 松尾研究室（情報通信工学科）における学生インターンシッププログラム（SIP）の一環として開発されました。

## 主な機能

- **環境モニタリング** — 気温・湿度・気圧、土壌温度・土壌湿度、照度、CO₂、空気質を、設定可能な間隔でサンプリングします。
- **自動制御** — ファン、スプレー、ミキサー、ライトの各リレーを、ヒステリシス・クールダウン・優先順位を考慮したしきい値ロジックで駆動します。ダッシュボードからの手動操作も可能です。
- **コンピュータビジョン（YOLOv8）** — 2 つの ONNX モデルを使用します。1 つは野菜くずの検出（`carrot_peels`、`mixed_veg`、`onion`）、もう 1 つはコンポストの熟成段階の分類（`raw`、`middle`、`mature`）を行います。
- **C/N 比の推定** — 検出結果をクラスごとの質量推定値（バウンディングボックス面積 × 充填レベルごとのグラム定数）に変換し、質量加重の C/N 比を計算します。炭素分の多い「ブラウン材」をどれだけ追加すべきかの推奨値も表示します。
- **ライブダッシュボード** — ダークテーマのカード型 CustomTkinter UI。ステータスピル、センサーごとのステータステキスト、トレンドスパークライン（CO₂ と土壌温度）、ライブカメラビュー、推定重量の内訳を表示します。
- **クラウド + ローカルログ** — 測定値は平均化されて Neon PostgreSQL データベースにアップロードされ、ローカルログにも記録されます。オプションで Grafana による可視化と Telegram アラートにも対応しています。

## ハードウェア

- Raspberry Pi（カメラモジュール付き）
- **BME280** — 気温・湿度・気圧（I²C）
- **SHT35** — 土壌温度・土壌湿度（I²C）
- **TSL2561** — 環境光 / 照度（I²C）
- **MQ135** — 空気質ガスセンサー（**ADS1115** ADC 経由で読み取り）
- **MH-Z19C** — NDIR 方式 CO₂ センサー（UART）
- 4 チャンネル**リレーボード**（アクティブロー）— LED、ファン、スプレー、ミキサーを駆動

### GPIO ピンマップ（BCM）

| アクチュエータ | GPIO |
|----------|------|
| LED   | 25 |
| ファン   | 24 |
| スプレー | 23 |
| ミキサー | 22 |

> リレーは**アクティブロー**です（`0` = ON、`1` = OFF）。すべてのピンは、起動時およびシャットダウン時に HIGH（OFF）に設定されます。

## ソフトウェアのセットアップ

### 前提条件

- Raspberry Pi OS（`raspi-config` で I²C、SPI、シリアルポートを有効化）
- Python 3.9 以上
- `pigpio` デーモンの起動:

```bash
sudo pigpiod
```

### インストール

```bash
git clone https://github.com/<your-username>/<your-repo>.git
cd <your-repo>
pip3 install -r requirements.txt
```

コードは主に以下をインポートします: `customtkinter`、`opencv-python`、`pillow`、`pigpio`、`onnxruntime`、`numpy`、および Neon 用の PostgreSQL クライアント。バージョンは `requirements.txt` で固定してください。ダッシュボードの画像ベースのアイコンには `customtkinter >= 5.0` が必要です。

## 設定

### 1. CO₂ センサー（MH-Z19C）

密閉された高 CO₂ 環境のコンポストビン内でベースラインがドリフトしないよう、自動校正（ABC）は起動のたびに無効化されます。初回使用前に、**コンポストから離れた場所で**、一度だけ新鮮空気ゼロ点校正を実行してください:

```bash
python3 calibrate_co2.py
```

プロンプトに従ってください（新鮮な空気中で約 20 分間ウォームアップした後、確認します）。詳細はスクリプトのヘッダーを参照してください。

### 2. 空気質センサー（MQ135）

クリーンエア時のベースライン抵抗値（`R0`）は JSON ファイルに保存され、実行時の `Rs/R0` の計算に使用されます。クリーンな空気中で校正スクリプトを実行してファイルを（再）生成し、`main.py` 内のパスがスクリプトの出力先と一致していることを確認してください。

### 3. データベース（Neon PostgreSQL）

Neon の接続情報は `Database_Neon.py` に設定するか、環境変数で指定してください（認証情報がバージョン管理に含まれないよう、環境変数の使用を推奨します）。

> **データベース URL、API キー、Telegram トークンはコミットしないでください。** `.env` ファイルまたは環境変数を使用し、`.gitignore` に追加してください。

## 使い方

```bash
sudo pigpiod        # 未起動の場合のみ、一度実行
python3 main.py
```

ダッシュボードでは以下の操作が可能です:

- ステータステキストと CO₂ / 土壌温度のトレンドスパークライン付きのライブセンサータイルを確認する。
- リレーを手動で切り替える、または自動制御ロジックとタイマー式ミキサーを実行する。
- 目標範囲とインターバルをリアルタイムで設定する（Enter またはフォーカスアウトで反映）。
- ライブビューでカメラの向きを調整し、**Capture Now** で検出を実行する。
- 検出されたクラスごとに充填レベルを調整し、再撮影せずに C/N 推定値を再計算する。

## AI モデルについて

両方の検出器は、デバイス上での推論のために ONNX にエクスポートされた YOLOv8 モデルです。

- **野菜検出:** `carrot_peels`、`mixed_veg`、`onion`
- **コンポスト段階:** `raw`、`middle`、`mature`

本プロジェクトの中心的な知見は、**モデルのアーキテクチャよりもラベルの品質のほうが重要である**という点です。YOLOv8n と YOLO11 を入れ替えても精度はほぼ同一であり、ボトルネックはネットワークではなくアノテーションの一貫性にあります。ラベリングは、クラスが支配的な領域ごとに 1 つのボックスを付け、野菜にぴったり沿わせて土壌部分を除外し、クラスが絡み合った曖昧な画像は無理にラベル付けせずスキップする方針で行っています。

学習、データセット分割、評価（mAP、混同行列）は `Compost_Model.ipynb` で行います。
